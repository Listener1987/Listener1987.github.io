    java异常类型

    Error
    java虚拟机无法解决的严重问题，JVM系统内部错误，资源耗尽等

Exception
因编程错误或偶然因素导致的问题，可修改代码进行处理,空指针访问，读取不存在文件，数组越界等

编译时异常(checked)

  • IOException
  • FileNotFoundException
  • ClassNotFoundException

    运行时异常(unchecked)

  • NullPointerException空指针
  • ArrayIndexOutOfBoundsERxception数组越界
  • ClassCastException类型转换异常
  • NumberFormatException数字格式异常
  • InputMismatchException输入不匹配
  • ArithmeticException算数异常

java异常处理机制

异常的处理：抓抛模型

过程一:抛，一旦出现异常，就会在异常代码处生成一个对应异常类的对象
并将此对象抛出，其后的代码也不再执行

过程二:抓,异常处理的方式。
* try-catch-finally
* throws

###try-catch-finally
说明

  • catch的异常类型如果有子父类关系，则子类在上，否则无所谓

  • 常用的异常对象处理的方式 String getMessage()或printStackTrace()

  • try结构声明的变量，不能在外部调用

  • finally是可选的

  • finally使用的情景，当出现catch中异常，或try，catch中有return语句时依然可以调用

  • finally使用的意义，数据库连接，输入输出流，网络编程中有些资源需要人为回收，当代码出现问题时，可在finally中声明资源关闭语句

    throws+异常类型

  • 并没有真正的处理异常，而是把异常交给调用者使用try-catch处理

  • throws使用的方向，几个方法层层递进，每个方法都有可能出错，则使用throws效果更好

    throw

  • throw可手动生成异常对象

  • 用户可自定义异常类型

###总结：

  • 子类重写的方法抛出的异常类型不大于父类被重写的方法抛出的异常类型
  • 若父类被重写的方法没有使用throws则不能用throws，只能用try-catch
